Eureka

–

1.Eureka Server

微服务主要是由两个组件构成 Eureka Server注册中心和Eureka Client服务端.

Eureka Server作为服务注册功能的服务器 它是服务注册中心而系统中其它微服务使用 Eureka Client 客户端 连接到Eureka Server 维持心跳连接这样就能监控系统中各个微服务是否正常运行.

2.Eureka Server 注册中心

2.1 创建新项目

2.2 配置pom

2.3 在启动类上添加 @EnableEurekaServer 表示有注册中心功能

eureka:  
    client:  
        service-url: 
            defaultZone: http://localhost:8761/eureka/
    不允许eureka成为注册中心
    register-with-eureka: false

3.Eureka Client服务注册

3.1 启动Eureka Server注册中心

项目打jar包             mvn clean package 
java运行 jar包         java -jar target/eureka-XXX.jar
后台运行eureka注册中心   nohup java -jar target/eureka-0.0.1-SNAPSHOT.jar > /dev/null 2>&1 &

3.2 创建eureka client-project项目

1.cloud discovery -> eureka discovery 2.在启动类上添加注解 @EnableDiscoveryClient 3.添加 application.yml配置文件

4.创建eureka server 高可用

在端口8761中如下配置文件 
    eureka:  
        client:  
            service-url: 
                defaultZone: http://localhost:8762/eureka/
        不允许eureka成为注册中心
        register-with-eureka: false

同样在client 配置中的defaultZone中添加 8761 和 8762 ，这样当 8762挂了 仍然可以访问到

5.eureka总结

1.@EnableEurekaServer 和 @EnableDiscoveryClient 2.心跳检测、健康检测、负载均衡 3.eureka 高可用，生产上建议至少2台

分布式系统中，服务注册中心是最重要的，最基础的功能

关于注册中心

注册中心 A在多个B找到一个B 这是客户端发现

使用代理 让代理找到一个B后交给A 这是服务端发现

客户端发现 优点：简单方便，知道所有B可用的地址 （eureka）缺点：A自己实现一套逻辑将B挑选出

服务端发现（Nginx，Zookeeper） 优点 A对于B是透明且不可见的
