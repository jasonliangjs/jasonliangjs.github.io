总结git一些命令

综述

  • git的时光机穿梭
  • git的远程仓库
  • git的分支管理

git的时光机穿梭-git常见命令

切换其他git远程仓库：

git remote rm origin   
git remote add origin git@github.com:jasonliangjs/learngit.git

1.选择一个合适位置创建一个空目录

mkdir/Users/michael/learngit

2.把目录变成Git可以管理的仓库

git init

掌握工作区状态

git status

如果git status显示有修改，查看修改内容

git diff

3.将代码（readme）从工作区添加到暂存区

git add readme.txt

4.‘注释’文件 将缓存区内容添加到本地仓库

git commit -m "append GPL"

5.显示git的日志，—-pretty=oneline 精简日志

git log --pretty=oneline

git的时光机

1.撤回上一个版本的git

git reset --hard HEAD^

2.回滚到相应的版本号对应的版本

git reset --hard  8ff33475c6797d10629ba849cf721bfe39ad863e

3.查看提交历史

git log

4.查看所有提交历史

git reflog

修改的情况

1.如果git并没有执行add ，也就是还在工作区

git checkout —- readme.txt

2.如果执行add，处于暂存区，先回到工作区，再重复1

git reset HEAD readme.txt

git checkout -—readme.txt

创建分支

1.创建分支(dev)

git checkout -b dev

2.查看当前分支

git branch

对master中的readme.txt进行修改  
git add readme.txt - git commit -m “branch test”提交

git checkout master切回master分支 ，查看master中的readme并无变化

3.合并分支

git merge dev 合并完成后删除dev分支，再次查看branch 只剩下master

4.删除分支

git branch -d dev

远程仓库

创建SSH Key

$ ssh-keygen -t rsa -C "youremail@example.com"

添加远程库

1.关联一个远程库

git remote add origin git@github.com:lwtcumt/test.git

2.关联后，使用命令第一次推送master分支的所有内容

git push -u origin master

3.关联后

git push origin master  即可推送

解决冲突

1.准备新的feature1分支

git checkout -b feature1

2.修改readme.txt并在分支上提交

git add readme.txt / git commit -m “and simple”

3.切换master 重复上面的操作,修改readme.txt

git checkout master

4.提交后会显示冲突 修改后保存

git add readme.txt / git commit -m ”conflict fixed”

5.git log 显示分支合并的情况，并删除分支

git log 
git branch -d feature1

6.合并分支的时候 使用的是Fast forward模式，git会在merge时生成一个新的commit，这样从历史上就能看出分支信息.

git merge  --no-ff -m “merge with no-ff dev

Bug分支(正在工作是临时修改bug的情况)

1.如果当前的程序正在进行中，需要搁置

git stash

2.修复后 git 回到工作现场

git stash pop

3.创建一个bug分支

git branch -b issue-101

4.修改readme.txt文件，并且提交

learngit git:(issue-101) ✗ git add readme.txt

learngit git:(issue-101) ✗ git commit -m "fix bug issue-101"

5.返回master分支并与issue-101合并

learngit git:(issue-101) git checkout master

git merge --no-ff -m "merged bug fix 101" issue-101

6.推送到github上

learngit git:(master) git push origin master

Feature分支(开发新功能-创建一个新分支feature)

1.开发一个新feature，最好新建一个分支

git branch -d feature-vulcan

2.如果要丢弃一个没有被合并过的分支

git branch -D feature-vulcan

多人协作

1.当clone从远程仓库的时候，实际上git自动把本地master和远程的master分支对应起来，并且远程仓库对应的是origin 　 查看远程库信息

git remote -v

2.推送分支 - 将该分支上所有的本地提交推送到远程库 git push origin master/dev(其他分支)

 git push origin master/dev(其他分支)

在抓取分支的时候 clone默认情况下只能看到本地master分支 - 希望在dev分支开发，那么必须创建远程的origin的dev分支到本地 　　　　　　　　　git checkout -b dev origin/dev

git checkout -b dev origin/dev

git add env.txt / 

git commit -m "add env" /git push origin dev

3.推送失败　　　　　　　　　　　　　　　　　

试图合并  git pull

4.合并冲突，解决，并在本地解决

5.没有冲突或者解决了冲突

git push origin 分支名
